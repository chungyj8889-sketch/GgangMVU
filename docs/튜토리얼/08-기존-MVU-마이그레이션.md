# 기존 MVU 마이그레이션

기존 MVU 설정을 MVU Zod로 마이그레이션하는 방법을 안내합니다.

## 마이그레이션 개요

기존 MVU를 사용하고 계셨다면, MVU Zod로 쉽게 마이그레이션할 수 있습니다. 대부분의 설정은 자동으로 변환되며, 일부 설정만 수동 조정이 필요합니다.

## 자동 마이그레이션

### 1. 백업 생성

!!! danger "중요"
    마이그레이션 전 반드시 기존 설정을 백업하세요!

```
설정 > 내보내기 > 전체 백업
```

### 2. 마이그레이션 도구 실행

```
도구 > MVU Zod 마이그레이션
```

### 3. 변환 결과 확인

마이그레이션 도구가 변환 결과를 보여줍니다:

| 항목 | 상태 |
|------|------|
| 변수 | ✅ 변환 완료 |
| 프롬프트 | ✅ 변환 완료 |
| 스크립트 | ⚠️ 수동 확인 필요 |

## 수동 마이그레이션

자동 변환이 불가능한 항목들:

### 커스텀 스크립트

기존 문법:
```javascript
// 기존 MVU
$변수명 = "값";
```

새로운 문법:
```javascript
// MVU Zod
setVariable("변수명", "값");
```

### 조건문 변환

기존:
```
[IF $상태 == "전투"]내용[/IF]
```

새로운:
```
{{#if (eq 상태 "전투")}}내용{{/if}}
```

## 호환성 모드

마이그레이션 기간 동안 호환성 모드를 사용할 수 있습니다:

```javascript
{
  "compatibilityMode": true,
  "legacySyntax": true
}
```

!!! warning "권장하지 않음"
    호환성 모드는 임시 해결책입니다. 가능한 빨리 새 문법으로 전환하세요.

## 문제 해결

??? question "변수가 인식되지 않아요"
    변수 이름에 특수문자가 포함되어 있는지 확인하세요. MVU Zod는 더 엄격한 명명 규칙을 사용합니다.

??? question "스크립트가 동작하지 않아요"
    함수 이름이 변경되었을 수 있습니다. 마이그레이션 가이드의 함수 대응표를 확인하세요.
